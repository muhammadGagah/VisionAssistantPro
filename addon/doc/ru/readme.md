# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый мультимодальный ИИ-ассистент для NVDA. Он использует первоклассные ИИ-движки для интеллектуального чтения экрана, перевода, голосового ввода и анализа документов.

_Это дополнение выпущено в честь Международного дня инвалидов._

## 1. Установка и настройка

Перейдите в **Меню NVDA > Параметры > Параметры > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый ИИ-сервис. Поддерживаемые провайдеры включают **Google Gemini**, **OpenAI**, **Mistral**, **Groq**, **MiniMax** и **Custom** (серверы, совместимые с OpenAI, такие как Ollama, LM Studio, Jan.ai или KoboldCPP).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений/файлов).
- **API-ключ:** Обязательное поле. Вы можете ввести несколько ключей (разделяя их запятыми или новыми строками) для автоматической ротации.
- **Получить модели:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить последний список доступных моделей от провайдера.
- **ИИ-модель:** Выберите основную модель, используемую для общего чата и анализа.

### 1.2 Расширенная маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Custom.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для продвинутых пользователей**. Если вы не уверены, что делает конкретная модель, оставьте этот параметр **отключенным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) вызовет ошибки и остановит работу дополнения.

Установите флажок **"Расширенная маршрутизация моделей (для конкретных задач)"**, чтобы получить детальный контроль. Это позволит вам выбирать конкретные модели из раскрывающегося списка для разных задач:
- **OCR / Vision модель:** Выберите специализированную модель для анализа изображений.
- **Распознавание речи (STT):** Выберите конкретную модель для диктовки.
- **Синтез речи (TTS):** Выберите модель для генерации аудио.
- **Модель AI Operator:** Выберите конкретную модель для задач автономного управления компьютером.
- **Видео-модель:** Выберите конкретную модель для анализа видео и генерации аудиоописаний.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Расширенная настройка конечных точек (Пользовательский провайдер)
*Доступно только при выборе "Custom".*

> **⚠️ Предупреждение:** Этот раздел позволяет вручную настраивать API и предназначен для **опытных пользователей**, запускающих локальные серверы или прокси. Неправильные URL-адреса или имена моделей нарушат подключение. Если вы точно не знаете, что это за конечные точки, оставьте этот параметр **отключенным**.

Установите флажок **"Расширенная конфигурация конечных точек"**, чтобы вручную вводить детали сервера. В отличие от стандартных провайдеров, здесь вы должны **ввести** конкретные URL-адреса и имена моделей:
- **URL-адрес списка моделей:** Конечная точка для получения доступных моделей.
- **URL-адреса конечных точек OCR/STT/TTS:** Полные URL-адреса для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Пользовательские модели:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.3.1 Настройка локального ИИ (Настройка в одно действие)
Чтобы упростить интеграцию с полностью локальным офлайн-ИИ, внутри настроек пользовательского провайдера доступна специальная кнопка **"Настроить локальный ИИ"**.

Если на вашем компьютере запущен локальный сервер ИИ-моделей:
1. Выберите **Custom** в качестве провайдера.
2. Нажмите кнопку **Настроить локальный ИИ**.
3. Выберите ваш локальный ИИ-движок из доступного диалога:
   - **Ollama** (по умолчанию `http://127.0.0.1:11434`)
   - **LM Studio** (по умолчанию `http://127.0.0.1:1234`)
   - **Jan.ai** (по умолчанию `http://127.0.0.1:1337`)
   - **KoboldCPP** (по умолчанию `http://127.0.0.1:5001`)
4. Дополнение мгновенно настроит правильный локальный URL, тип API и автоматически получит ваши активные офлайн-модели, чтобы заполнить поле выбора **"ИИ-модель"**.

*Примечание о сети и прокси:* Этот движок локального подключения имеет расширенный механизм обхода прокси. Даже если у вас активен системный VPN или прокси в режиме TUN, ваши локальные запросы к ИИ будут полностью обходить его, обеспечивая стабильные офлайн-подключения без ошибок 502 Bad Gateway.

### 1.4 Общие настройки
- **OCR движок:** Выберите между **Chrome (Быстрый)** для быстрых результатов или **AI (Расширенный)** для лучшего сохранения форматирования.
- **TTS голос:** Выберите предпочтительный стиль голоса. Этот список обновляется динамически в зависимости от вашего активного провайдера.
- **Креативность (Температура):** Управляет случайностью ИИ. Более низкие значения лучше подходят для точного перевода/OCR.
- **URL прокси:** Настройте, если ИИ-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, например `127.0.0.1` или мостовые URL).
- **Прямой вывод (Без окна чата):** Установите этот флажок, если хотите, чтобы ИИ просто зачитывал результат вслух, не открывая интерактивное окно чата.
- **Копировать ответы ИИ в буфер обмена:** Автоматически копирует каждый ответ ИИ в системный буфер обмена для удобной вставки.
- **Очищать разметку Markdown в чате:** Снимите этот флажок, если вы предпочитаете видеть необработанные символы форматирования вместо чистого форматированного текста.

1. Нажмите **NVDA + Shift + V** (Главная клавиша), чтобы активировать слой (вы услышите звуковой сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих клавиш:

## 3. AI Operator - Автономное управление компьютером

**AI Operator** превращает Vision Assistant Pro из пассивного средства чтения в активного помощника, который может взаимодействовать с вашим компьютером от вашего имени. Вы можете попросить его описать экран, ответить на вопросы о том, что он видит, или даже взять управление на себя — нажимать кнопки, перетаскивать элементы, вводить текст и перемещаться по приложениям с помощью команд на естественном языке.

Самое большое преимущество? Он отлично работает в полностью недоступном программном обеспечении. Если вы застряли в пользовательском приложении, на удаленном рабочем столе или на сайте, где ваша программа чтения с экрана полностью молчит, оператору это не мешает. Поскольку он "видит" экран визуально, он может находить, читать и взаимодействовать с элементами, у которых нет меток доступности.

### Как это работает
1. Нажмите **NVDA + Shift + V**, затем нажмите **Shift + A** (или используйте прямую горячую клавишу), чтобы открыть диалог AI Operator.
2. Введите простым языком, что вы хотите сделать (например, "Нажми кнопку Сохранить", "Что говорит сообщение об ошибке?" или "Переименуй файл в final.pdf").
3. ИИ проанализирует ваш экран, определит соответствующие элементы и выполнит действие или предоставит ответ. Если задача требует нескольких шагов, оператор будет продолжать работу до ее завершения.
4. Нажмите **Shift + A** снова в любое время, чтобы мгновенно прервать выполняемую операцию.

### Поддерживаемые действия
Оператор понимает широкий спектр команд:
- **Опиши и ответь**: "Опиши макет экрана" или "Что говорит сообщение об ошибке?"
- **Нажми**: "Нажми кнопку Сохранить"
- **Правая кнопка мыши**: "Щелкни правой кнопкой мыши по файлу"
- **Двойной щелчок**: "Дважды щелкни по документу"
- **Перетащи**: "Перетащи документ в папку Архив"
- **Введи**: "Введи 'Привет, мир' в поле поиска"
- **Прокрутка**: "Прокрути вниз три раза"
- **Нажатие клавиш**: "Нажми Enter", "Нажми Tab", "Нажми Escape"
- **Многошаговые задачи**: "Открой Проводник, найди отчет и переименуй его в final.pdf"

### Важные примечания
- **⚠️ Предупреждение об использовании API**: Поскольку оператору необходимо точно "видеть", что происходит на экране, он отправляет скриншот высокого разрешения на каждом шаге. Частое использование будет расходовать вашу квоту API гораздо быстрее, чем стандартные текстовые функции.
- **Приложения с правами администратора**: Если NVDA запущена без прав администратора, оператор может не иметь возможности взаимодействовать с окнами, требующими повышенных привилегий. Это ограничение безопасности Windows, а не ошибка дополнения.
- **Рекомендации**: Для достижения наилучших результатов давайте четкие и конкретные команды. "Нажми синюю кнопку Отправить внизу формы" почти всегда будет работать лучше, чем просто "Нажми кнопку".

## 4. Видеоанализ и аудиоописание

> **Примечание:** Функции видеоанализа и аудиоописания строго работают на провайдере **Google Gemini**. Убедитесь, что в настройках дополнения в качестве активного провайдера выбран Google Gemini.

Vision Assistant Pro представляет мощные возможности обработки видео, разработанные специально для незрячих пользователей. Он может анализировать как онлайн-видео, так и локальные записи экрана, предоставляя очень подробные визуальные описания и генерируя профессиональные скрипты аудиоописания (SRT).

### 4.1 Локальная запись экрана (Control + V)
Если вы столкнулись с немым видео, анимацией или обучающим роликом на экране, вы можете захватить его напрямую:
1. Нажмите **NVDA + Shift + V** для входа в Командный слой, затем нажмите **Control + V**.
2. Дополнение начнет фоновую запись вашего экрана.
3. Нажмите **Control + V** снова, чтобы остановить запись.
4. Затем ИИ проанализирует записанный сегмент видео и предоставит очень подробное описание сцены, персонажей и действий.

### 4.2 Видеоанализ (Shift + V)
Вы можете анализировать как локальные видеофайлы, так и онлайн-видео. Просто выберите локальный видеофайл в Проводнике Windows или скопируйте ссылку на онлайн-видео в буфер обмена. Вы также можете нажать **Shift + V** в любом месте (например, внутри медиаплеера), чтобы открыть диалог, где можно выбрать видеофайл или вручную вставить URL.
- **Поддерживаемые онлайн-платформы:** YouTube, Instagram, TikTok и Twitter (X).
- ИИ автоматически определит локальный файл или URL, обработает видео и предоставит всестороннее визуальное описание и аудиосводку.

### 4.3 Генерация аудиоописания (SRT)
Для более структурированного опыта дополнение может генерировать профессиональные скрипты аудиоописания в стандартном формате SubRip (SRT).
- **Умная расстановка пауз:** ИИ прослушивает аудиодорожку и специально привязывает свои визуальные описания к естественным паузам и тихим промежуткам, чтобы разумно минимизировать перекрытие с диалогом.
- **Отслеживание персонажей:** Движок выполняет предварительный проход для выделения различных персонажей на основе неизменных черт лица. Он создает глобальный словарь для точного отслеживания и маркировки персонажей в разных сценах без путаницы.
- **Дословный OCR текста:** Любой текст, появляющийся на экране (вывески, телефоны, титры), строго цитируется дословно.
- **Как использовать:** Чтобы прослушать сгенерированные субтитры, просто поместите файл `.srt` в ту же папку, что и ваш видеофайл, и дайте ему точно такое же имя. Затем настройте свой медиаплеер (например, VLC или PotPlayer) на маршрутизацию текста субтитров непосредственно в вашу программу чтения с экрана или TTS-движок во время воспроизведения.

### 4.4 Синхронизированное аудио-озвучивание (Экспорт MP3)
Помимо создания текстовых SRT-файлов, дополнение функционирует как полноценный инструмент для создания аудиоописаний, синтезируя описания в речь и смешивая их с видео. При генерации MP3 для локальных видеофайлов доступно несколько режимов микширования:
- **Стандартное AD (смешивание голоса):** Озвучивание накладывается непосредственно поверх аудио видео. Вам будет предложено применить **Audio Ducking** (понижение громкости фона во время описаний), чтобы обеспечить четкость озвучивания.
- **Расширенное AD (пауза аудио):** Движок приостанавливает исходное аудио видео во время описаний, гарантируя, что вы не пропустите ни слова из исходного диалога или озвучивания ИИ.
- **YouTube видео:** Для источников YouTube (которые не загружаются локально) экспорт MP3 будет строго содержать синхронизированную голосовую дорожку ИИ без фонового аудио видео.

## 5. Расширенное чтение документов и изображений

Vision Assistant Pro включает высокооптимизированное средство чтения документов, предназначенное для многостраничных PDF-файлов, сложных изображений и даже форматов iPhone HEIC.

### 5.1 Пакетная обработка и возобновление
Вам не нужно читать огромный документ за один раз. Введите диапазон страниц (например, `1-20`), и ИИ обработает все страницы в фоновом режиме. Если NVDA зависнет или вы прервете сканирование, дополнение запомнит ваш прогресс и предложит **Возобновить** ровно с того места, где вы остановились!

### 5.2 Умное действие с файлом
Вам не всегда нужно сначала открывать документ. В Проводнике Windows просто выделите PDF или изображение и нажмите **D** (Чтение документов) или **F** (Умное действие с файлом) внутри Командного слоя. Дополнение мгновенно обойдет диалог выбора файла и начнет обработку выделенного файла.

### 5.3 Горячие клавиши просмотра документов
Когда открыто окно средства чтения документов, вы можете использовать следующие горячие клавиши:
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть диалог чата, чтобы задать вопросы о документе.
- **Alt + R:** Принудительно **Пересканировать с помощью ИИ**, используя вашего активного провайдера.
- **Alt + G:** Сгенерировать и сохранить высококачественный аудиофайл (WAV/MP3). *(Скрыто, если провайдер не поддерживает TTS).*
- **Alt + S / Ctrl + S:** Сохранить извлеченный текст как TXT или HTML файл.

## 6. Семантическая ИИ-маркировка и UI Explorer

Застряли в приложении с повсюду "неподписанными кнопками"? Движок семантической ИИ-маркировки решает эту проблему навсегда.

### 6.1 Постоянная маркировка объектов (L)
Сфокусируйте вашу программу чтения с экрана на неподписанном графическом элементе или кнопке и нажмите **L** в Командном слое. ИИ посмотрит на кнопку визуально, определит ее функцию и применит постоянную метку.
*В отличие от старых инструментов маркировки программ чтения с экрана, это дополнение использует продвинутую гибридную систему "Подписи объекта" (AutomationId/ControlID). Ваши пользовательские метки переживут изменение размера окна, смену монитора и обновления приложений!*

### 6.2 Полное сканирование приложения (Shift + L)
Нажмите **Shift + L**, чтобы сканировать все активное окно сразу. ИИ найдет все неподписанные элементы и разумно назовет их за один раз. Позже вы сможете управлять, переименовывать или массово удалять эти метки из встроенного Менеджера меток.

### 6.3 UI Explorer (E)
Нужно взаимодействовать с элементом, не перемещаясь к нему вручную? Нажмите **E**, чтобы активировать UI Explorer. ИИ просканирует экран и сгенерирует доступный список всех кликабельных элементов (игнорируя системный шум, такой как панели задач). Выберите элемент из списка, и дополнение мгновенно щелкнет по нему за вас.

## 7. Живой голосовой ассистент

Живой ассистент превращает Vision Assistant Pro в интерактивного помощника в реальном времени.
*(Примечание: Эта функция доступна только для Google Gemini и совместимых с Gemini пользовательских провайдеров).*

- **Активация:** Нажмите **Control + L** в Командном слое, чтобы открыть диалог Живого ассистента.

## 9. Примеры использования в реальной жизни (Какую функцию мне использовать?)

Vision Assistant Pro полон продвинутых инструментов. Вот несколько распространенных сценариев, которые помогут вам выбрать правильный:

- **Сценарий: Вы хотите понять полную компоновку сложного окна или недоступного приложения.**
  *Решение:* Нажмите **O** (Full Screen Vision). ИИ проанализирует весь экран и опишет, где именно расположены элементы, тексты и кнопки.

- **Сценарий: Вы нашли изображение на веб-странице или неподписанный графический элемент в документе.**
  *Решение:* Переместите объект навигатора на графический элемент и нажмите **V** (Object Vision). ИИ опишет именно то, что содержит это изображение.

- **Сценарий: Вы хотите посмотреть фильм или видеоклип с аудиоописаниями.**
  *Решение:* Нажмите **Shift + V** на вашем видео и выберите **"Сгенерировать аудиоописание (SRT файл)"**. Когда он завершится, нажмите **"Сгенерировать синхронизированное озвучивание (MP3)"** и выберите **"Расширенное AD"**. Дополнение создаст аудиодорожку, которая интеллектуально приостанавливает диалог фильма для описания визуальных сцен.

- **Сценарий: Вы столкнулись с приложением, полным "неподписанных кнопок".**
  *Решение:* Нажмите **L**, чтобы навсегда пометить конкретную кнопку с помощью ИИ. Или нажмите **Shift + L**, чтобы отсканировать и пометить все окно сразу. Если вы просто хотите быстро нажать на что-то, нажмите **E** (UI Explorer), чтобы получить список всех кликабельных элементов.

- **Сценарий: Вам нужно обойти недоступную CAPTCHA.**
  *Решение:* Нажмите **C** (Решение CAPTCHA). ИИ автоматически захватит CAPTCHA, решит ее и вставит ответ в правильное поле.

- **Сценарий: Вы хотите прочитать длинный PDF-документ на 50 страниц.**
  *Решение:* Нажмите **D** (Чтение документов), установите провайдера Google Gemini и введите диапазон страниц `1-50`. Дополнение точно извлечет текст в фоновом режиме.

- **Сценарий: Вы смотрите немой видеоурок или анимацию на экране.**
  *Решение:* Нажмите **Control + V**, чтобы начать запись экрана. Дайте уроку воспроизвестись, затем снова нажмите **Control + V**. ИИ объяснит, что именно было продемонстрировано.

***
**Примечание:** Для всех функций ИИ требуется активное интернет-соединение. Многостраничные документы обрабатываются автоматически.

*   **Отчет о состоянии:** Добавлена новая команда (NVDA+Control+Shift+I) для объявления текущего состояния дополнения (например, "Загрузка...", "Анализ...").
*   **Экспорт в HTML:** Кнопка "Сохранить содержимое" в диалогах результатов теперь сохраняет вывод в виде форматированного HTML-файла, сохраняя стили, такие как заголовки и жирный текст.
*   **Интерфейс настроек:** Улучшен макет панели настроек с доступной группировкой.
*   **Новые модели:** Добавлена поддержка gemini-flash-latest и gemini-flash-lite-latest.
*   **Языки:** Добавлен непальский в список поддерживаемых языков.
*   **Логика меню "Редактировать":** Исправлена критическая ошибка, из-за которой команды "Редактировать текст" не работали, если языком интерфейса NVDA был не английский.
*   **Диктовка:** Улучшено обнаружение тишины для предотвращения некорректного вывода текста при отсутствии речевого ввода.
*   **Настройки обновления:** "Проверять обновления при запуске" теперь отключено по умолчанию в соответствии с политикой Магазина дополнений.
*   Очистка кода.
