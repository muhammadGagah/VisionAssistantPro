# Vision Assistant Pro 文档

**Vision Assistant Pro** 是专为 NVDA 设计的高级多模态 AI 助手。它利用世界顶级的 AI 引擎提供智能屏幕阅读、翻译、语音听写和文档分析功能。

_本插件的发布旨在纪念国际残疾人日，以此回馈社区。_

## 1. 设置与配置

请前往 **NVDA 菜单 > 偏好设置 > 设置 > Vision Assistant Pro**。

### 1.1 连接设置
- **提供商：** 选择您首选的 AI 服务。支持的提供商包括 **Google Gemini**、**OpenAI**、**Mistral**、**Groq** 以及 **自定义**（兼容 OpenAI 的服务器，如 Ollama/LM Studio）。
- **重要提示：** 我们强烈建议使用 **Google Gemini** 以获得最佳的性能和准确度（尤其是在图像/文件分析方面）。
- **API 密钥：** 必填项。您可以输入多个密钥（用逗号或换行符分隔），系统会自动进行轮换以避免触发限制。
- **获取模型：** 输入 API 密钥后，按此按钮可直接从提供商处下载最新的可用模型列表。
- **AI 模型：** 选择用于常规聊天和分析的主模型。

### 1.2 高级模型路由（原生提供商）
*适用于 Gemini、OpenAI、Groq 和 Mistral。*

> **⚠️ 警告：** 这些设置**仅供高级用户使用**。如果您不确定某个特定模型的作用，请**取消勾选**此项。为某项任务选择了不兼容的模型（例如将纯文本模型用于视觉分析）会导致报错并使插件无法工作。

勾选**“高级模型路由（特定任务）”**可解锁详细控制。这允许您从下拉列表中为不同的任务选择特定的模型：
- **OCR / 视觉模型：** 选择用于分析图像的专用模型。
- **语音转文本 (STT) 模型：** 选择用于语音听写的特定模型。
- **文本转语音 (TTS) 模型：** 选择用于生成音频的模型。
*注意：不支持的功能（例如 Groq 的 TTS）将被自动隐藏。*

### 1.3 高级端点配置（自定义提供商）
*仅在选择了“自定义”提供商时可用。*

> **⚠️ 警告：** 此部分允许手动配置 API，专为运行本地服务器或代理的**专家用户**设计。错误的 URL 或模型名称将导致连接失败。如果您不完全清楚这些端点是什么，请保持此项**取消勾选**。

勾选**“高级端点配置”**以手动输入服务器详细信息。与原生提供商不同，在这里您必须**手动输入**特定的 URL 和模型名称：
- **模型列表 URL：** 获取可用模型的端点。
- **OCR/STT/TTS 端点 URL：** 特定服务的完整 URL（例如 `http://localhost:11434/v1/audio/speech`）。
- **自定义模型：** 为每项任务手动输入模型名称（例如 `llama3:8b`）。

### 1.4 常规偏好设置
- **OCR 引擎：** 在 **Chrome (快速)**（获取快速结果）或 **AI (高级)**（出色的布局保留和表格识别）之间选择。
    - *注意：* 如果您选择了“AI (高级)”但您的提供商设置为 OpenAI/Groq，插件将智能地将图像路由到您当前活动提供商的视觉模型。
- **TTS 语音：** 选择您喜欢的语音风格。此列表会根据您当前的活动提供商动态更新。
- **创造力 (温度)：** 控制 AI 的随机性。较低的值更有利于准确的翻译/OCR。
- **代理 URL：** 如果 AI 服务在您所在地区受限，请配置此项（支持本地代理如 `127.0.0.1` 或桥接 URL）。

## 2. 命令层与快捷键

为防止键盘冲突，本插件使用**命令层**。
1. 按 **NVDA + Shift + V**（主键）激活该层（您会听到“嘟”的一声）。
2. 松开按键，然后按以下单键之一：

| 按键          | 功能                     | 描述                                                                        |
|---------------|--------------------------|-----------------------------------------------------------------------------|
| **T**         | 智能翻译                 | 翻译导航对象光标下的文本或选中文本。                                        |
| **Shift + T** | 剪贴板翻译               | 翻译当前剪贴板中的内容。                                                    |
| **R**         | 文本润色                 | 摘要、语法修复、解释或运行**自定义提示词**。                                |
| **V**         | 对象视觉                 | 描述当前导航对象。                                                          |
| **O**         | 全屏视觉                 | 分析整个屏幕布局和内容。                                                    |
| **Shift + V** | 在线视频分析             | 通过 URL 分析 **YouTube**、**Instagram**、**TikTok** 或 **Twitter (X)** 视频。|
| **D**         | 文档阅读器               | 具有页面范围选择功能的高级 PDF 和图像阅读器。                               |
| **F**         | 文件 OCR                 | 直接从选定的图像、PDF 或 TIFF 文件中识别文本。                              |
| **A**         | 音频转录                 | 将 MP3、WAV 或 OGG 文件转录为文本。                                         |
| **C**         | 验证码求解器             | 捕获并解析验证码（支持政务门户等）。                                        |
| **S**         | 智能听写                 | 将语音转换为文本。按一次开始录音，再按一次停止/输入。                       |
| **L**         | 状态报告                 | 播报当前进度（例如“正在扫描...”、“空闲”）。                                 |
| **U**         | 检查更新                 | 手动检查 GitHub 以获取插件的最新版本。                                      |
| **Space**     | 调用上一次结果           | 在聊天对话框中显示上一次 AI 回复，以便查看或进行后续提问。                  |
| **H**         | 命令帮助                 | 显示命令层内所有可用快捷键的列表。                                          |

### 2.1 文档阅读器快捷键（阅读器内部）
通过 **D** 命令打开文档后：
- **Ctrl + PageDown：** 移至下一页。
- **Ctrl + PageUp：** 移至上一页。
- **Alt + A：** 打开聊天对话框，就文档内容进行提问。
- **Alt + R：** 强制**使用 AI 重新扫描**（使用您当前活动的提供商）。
- **Alt + G：** 生成并保存高质量音频文件 (WAV/MP3)。*如果提供商不支持 TTS，则隐藏此选项。*
- **Alt + S / Ctrl + S：** 将提取的文本保存为 TXT 或 HTML 文件。

## 3. 自定义提示词与变量

您可以在 **设置 > 提示词 > 管理提示词...** 中管理提示词。

### 支持的变量
- `[selection]`：当前选中的文本。
- `[clipboard]`：剪贴板内容。
- `[screen_obj]`：导航对象的截图。
- `[screen_full]`：全屏截图。
- `[file_ocr]`：选择用于提取文本的图像/PDF文件。
- `[file_read]`：选择用于阅读的文档 (TXT, 代码, PDF)。
- `[file_audio]`：选择用于分析的音频文件 (MP3, WAV, OGG)。

***
**注意：** 所有 AI 功能都需要有效的互联网连接。多页文档会自动处理。

* **多提供商架构**：在 Google Gemini 的基础上，全面添加了对 **OpenAI**、**Groq** 和 **Mistral** 的支持。用户现在可以选择自己首选的 AI 后端。
